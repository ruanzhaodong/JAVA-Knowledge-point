成员内部类的基本概念

定义

成员内部类是定义在另一个类的内部，作为外部类的一个成员存在的类。它可以像外部类的其他成员（如字段、方法）一样，被访问修饰符（如 public、private、protected 等）修饰。以下是成员内部类的基本语法结构：

        class OuterClass {
            // 外部类的成员变量和方法
            private int outerVariable;
        
            public void outerMethod() {
                System.out.println("这是外部类的方法");
            }
        
            // 成员内部类
            class InnerClass {
                // 内部类的成员变量和方法
                private int innerVariable;
        
                public void innerMethod() {
                    System.out.println("这是内部类的方法");
                }
            }
        }

特点

  1. 可以访问外部类的成员：成员内部类可以直接访问外部类的所有成员，包括私有成员。这是因为成员内部类持有一个对外部类对象的引用。例如，在内部类的方法中可以直接使用外部类的成员变量和调用外部类的方法。
  2. 依赖外部类对象：要创建成员内部类的对象，必须先创建外部类的对象。因为成员内部类是外部类的一个成员，它的存在依赖于外部类对象。
  3. 可以被访问修饰符修饰：成员内部类可以使用各种访问修饰符来控制其可见性，如 public 表示可以被任何类访问，private 表示只能在外部类内部访问。

创建成员内部类对象的方式

要创建成员内部类的对象，需要先创建外部类的对象，然后通过外部类对象来创建内部类对象。示例代码如下：

        public class Main {
            public static void main(String[] args) {
                // 创建外部类对象
                OuterClass outer = new OuterClass();
                // 通过外部类对象创建内部类对象
                OuterClass.InnerClass inner = outer.new InnerClass();
                // 调用内部类的方法
                inner.innerMethod();
            }
        }

作用

  1. 封装性：将相关的类组织在一起，提高代码的封装性。内部类可以访问外部类的私有成员，同时又可以将内部类的实现细节隐藏起来，外部类以外的其他类无法直接访问内部类。
  2. 实现多继承的效果：在 Java 中，一个类只能继承一个父类，但可以通过成员内部类来实现类似多继承的功能。一个类可以有多个成员内部类，每个内部类可以继承不同的类，从而在一定程度上弥补了 Java 单继承的局限性。
