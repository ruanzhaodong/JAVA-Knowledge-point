Java异常处理 - 异常抛出（throw关键字）

throw的用法

用于在方法内部手动抛出异常实例，常用于业务逻辑中检测到错误时主动中断流程，格式为：

if (条件不满足) {
    throw new 异常类型("异常描述信息");
}

典型应用场景

  • 参数校验：如方法要求参数非空时，检测到null值则抛出NullPointerException
  • 业务规则中断：如转账时余额不足，抛出BalanceNotEnoughException（自定义异常）
  • 包装底层异常：将第三方库的异常转换为自定义异常，统一异常处理逻辑
throw示例
public void transfer(double amount, double balance) {
    if (amount > balance) {
        // 手动抛出余额不足异常
        throw new IllegalArgumentException("转账金额(" + amount + 
                                           ")超过余额(" + balance + ")");
    }
    // 正常转账逻辑
}
调用方处理
try {
    transfer(1000, 500);
} catch (IllegalArgumentException e) {
    System.out.println("转账失败：" + e.getMessage());
    // 处理异常（如提示用户）
}

throw关键字 与 throws关键字的对比

throw

作用位置： 方法内部

核心功能： 手动抛出异常实例

语法格式： throw new 异常类型();

示例：

if (param == null) {
    throw new NullPointerException("参数不能为null");
}
throws

作用位置： 方法声明后

核心功能： 声明方法可能抛出的异常类型

语法格式： public void method() throws 异常类型;

示例：

public void readFile() throws FileNotFoundException {
    // 文件读取代码
}
