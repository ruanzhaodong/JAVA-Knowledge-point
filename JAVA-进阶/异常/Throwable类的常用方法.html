Java异常处理 - Throwable类的常用方法

1. String getMessage()

返回异常的详细描述信息，通常用于记录异常发生的具体原因，例如：

try {
    int result = 10 / 0;
} catch (ArithmeticException e) {
    System.out.println(e.getMessage()); // 输出 "除数不能为零"
}

2. void printStackTrace()

打印异常的堆栈跟踪信息，包括异常类型、错误信息及发生位置，输出格式如下：

java.lang.ArithmeticException: 除数不能为零
    at com.example.ExceptionDemo.main(ExceptionDemo.java:5)

补充方法：

  • StackTraceElement[] getStackTrace()：获取堆栈跟踪元素数组，可手动遍历堆栈信息
  • Throwable getCause()：获取导致当前异常的根本原因（因果链上游异常）
  • Throwable fillInStackTrace()：重新填充堆栈跟踪信息，常用于包装异常时
