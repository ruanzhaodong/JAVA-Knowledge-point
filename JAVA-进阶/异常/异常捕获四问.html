Java异常处理 - 捕获异常四问

1. 若try块中代码未出现异常，程序如何执行？

答：跳过所有catch块，若存在finally块则先执行finally中的代码，然后继续执行try-catch-finally结构后的后续代码。

2. 若try块中存在多个异常点，程序如何执行？

答：需编写多个catch块分别捕获不同类型的异常，且catch块需按“子类异常在前、父类异常在后”的顺序排列，避免子类异常被父类异常提前捕获。

try {
    // 可能抛出IOException或SQLException的代码
} catch (SQLException e) { // 先捕获子类异常
    e.printStackTrace();
} catch (IOException e) { // 再捕获父类异常
    e.printStackTrace();
}

3. 若try块中出现异常但未被catch捕获，程序如何运行？

答：异常会向上抛给JVM虚拟机处理，此时JVM会打印异常堆栈信息并终止当前线程（若为主线程则程序结束）。

4. 若try块中出现异常，异常代码下方的代码是否会执行？

答：不会执行。当try块中某行代码抛出异常时，后续代码会被跳过，程序立即跳转到对应的catch块进行异常处理。

finally块的特殊情况：

  • 即使try或catch块中存在return语句，finally块仍会在return前执行
  • 若try块中调用System.exit(0)，finally块将不会执行
